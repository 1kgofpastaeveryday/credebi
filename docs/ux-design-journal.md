# UX Design Journal

## 2026-03-21: 残額フォーカス (Spendable Balance) の導入

**Decision**: ダッシュボードのヒーロー数値を「給料日前日の予測残高」から「あと自由に使える ¥XX,XXX」（自由残額）に刷新。日割り予算は復活させず、給料日までの余裕額を1つの数字で表示する。

**User problem**: 予測残高は「こうなる見込み」という事実提示だが、ユーザーが本当に知りたいのは「今いくら使えるか」。TapSheetなどの競合アプリは `収入 - 支出 - 貯蓄 = 残額` というシンプルな残額表示で、アプリを開くたびに一目で状況が分かるUXを実現している。Credebiの予測エンジンはより精密だが、その出力を「自由残額」というユーザーが直感的に理解できる形で表面化する必要があった。

**Alternatives considered**:
1. 日割り予算の復活（¥2,000/日） → 却下。大学生の支出パターンは日によって大きく変動し、均等割りは非現実的（PROJ-R4-006の判断を維持）
2. 予測残高をそのまま表示（現状維持） → 却下。「¥38,200」が良いのか悪いのかの文脈がない
3. 自由残額 + SAFE/WARNING色セマンティクス → 採用。数字と状態を一画面で

**Evidence**: TapSheet (2026年1月リリース) の残額フォーカスUXがInstagram/TikTokで「初めて続いた家計簿」として支持されている。シンプルな1数字表示がユーザーの日常確認習慣を生んでいる。

**Outcome**: ダッシュボードのヒーローカードを「あと自由に使える ¥XX,XXX」に統一。給料日・最低残高は補助情報としてカード内に表示。SAFE/WARNING/CRITICALの色がカード全体に反映され、staleデータ時はオレンジ帯で劣化表示。Design Principle #3（安全側）により、変動費は過去3ヶ月平均で楽観しない。
